# Research Explained — Agent Instructions

## Video Production

When producing multiple HyperFrames videos (3+):
1. Build ONE video first as a prototype
2. Present it for review
3. Iterate until approved
4. Use the approved version as the explicit quality reference in prompts for the remaining videos
5. Only then parallelize the rest

Do not mass-produce all videos in parallel on the first attempt.

## Self-Review Before Presenting

After building any deliverable, review it against `visual-style.md` and the quality standards BEFORE showing it to the user. If you can identify problems, fix them first. Don't present work you know is subpar and wait for the user to catch it.

Check for: misleading data representations, boring/static layouts, thin/undersized elements, missing visual depth, redundant labels, unbalanced timing, lack of narrative arc, text overlap, font sizes below 16px, unbalanced spacing, too much dead space, elements not centered in their containers, and blocks that don't fill enough of the frame.

## Visual Style

Follow `visual-style.md` in the project root for palette, font sizes, and content guidelines when creating any HyperFrames composition.
